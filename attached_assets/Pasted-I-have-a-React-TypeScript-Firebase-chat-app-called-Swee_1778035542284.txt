## SUMMARY: WHO DOES WHAT (SYNC RULES)
## ─────────────────────────────────────
Follow this pattern in EVERY game to avoid race conditions:

  ┌─────────────────────────────────────────────────────┐
  │  isPlayer1 (uid < partnerUid) ALWAYS:               │
  │  • Creates new rounds / writes initial Firestore doc │
  │  • Triggers reveal (writes pandaComment to Firestore)│
  │  • Calls onSendToChat() once                         │
  │                                                      │
  │  isPlayer2 ALWAYS:                                   │
  │  • Waits for round to appear via onSnapshot           │
  │  • Reads reveal from Firestore (never triggers it)   │
  │  • Shows same result from Firestore                  │
  │                                                      │
  │  BOTH players:                                       │
  │  • Submit their own answers/picks directly           │
  │  • Subscribe to same Firestore doc                   │
  │  • See identical final state                         │
  └─────────────────────────────────────────────────────┘

This single rule eliminates ALL race conditions across all games.
